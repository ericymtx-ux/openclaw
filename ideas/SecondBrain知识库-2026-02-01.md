# Second Brain - 持久化知识库 - 2026-02-01

## 原文

**核心概念**：大多数人与 AI 的对话是一次性的消耗品——聊完即弃。然而，这些对话中往往包含了绝佳的创意、决策过程和知识总结。"Second Brain" 用例旨在通过 Next.js 构建一个持久化的知识库，自动捕获这些价值。

**核心架构**：
1. **前端**：Next.js 应用，提供类似于 Obsidian + Linear 的现代化 UI
2. **存储**：本地 Markdown 文件或 Headless CMS
3. **自动化层**：ClawdBot 的 "记忆技能（Memory Skill）"

**运作流程**：
当您与 Bot 聊天时，它会实时判断内容价值。如果识别到重要概念，它会自动在后台创建一个 Markdown 文件，并写入结构化的内容，而不是仅仅保存在聊天记录里。

## 总结

这是一个 **AI 对话知识沉淀系统**，将一次性对话转化为持久化、可检索的知识资产。

**核心价值**：
- **防止知识流失**：最佳创意、决策过程不会随对话消失
- **可检索**：建立结构化标签体系，便于后续查找
- **自动分类**：AI 自动判断内容价值并分类
- **持续积累**：每日对话自动沉淀，长期形成知识资产

**与现有系统的关系**：
- 补充 `memory/` 目录的每日记忆
- 超越 `ideas/` 的灵感记录
- 类似于 `TODO/` 的结构化，但更侧重知识而非任务

## 信息架构与标签体系

预设标签分类，AI 自动分类：

| 标签 (Tag) | 用途描述 | 触发示例 |
|------------|----------|----------|
| `Journal` | 每日流水账，记录当天的所有对话摘要 | 每天结束时自动生成 |
| `Notes` | 具体知识点的深度解析 | "解释一下 React Server Components 的原理..." |
| `Projects` | 项目相关的规划、待办与进度 | "帮我规划一下新视频的脚本..." |
| `Scripts` | 视频脚本、文章草稿等创作内容 | "写一段关于 AI 安全的开场白..." |
| `Decisions` | 重要决策及其理由 | 技术选型、业务方向等 |
| `Archive` | 归档内容，不再活跃但有参考价值 | 历史版本、废弃方案 |

### 3. 自动化层（Memory Skill）

**判断逻辑**：
```python
def should_capture(content):
    # 1. 内容长度阈值（超过 500 字）
    # 2. 包含关键决策词（"决定"、"选择"、"采用"）
    # 3. 包含知识密集词（"原理"、"机制"、"架构"）
    # 4. 用户明确标记（"记住这个"、"保存"）
    return score > threshold
```

**自动分类**：
```python
def classify(content):
    # 基于关键词和上下文分类
    if "为什么"、"解释"、"原理" in content:
        return "Notes"
    elif "计划"、"规划"、"待办" in content:
        return "Projects"
    elif "脚本"、"开场白"、"草稿" in content:
        return "Scripts"
    elif "决定"、"选择"、"采用" in content:
        return "Decisions"
    else:
        return "Journal"
```
