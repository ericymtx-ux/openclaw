# Monday Dashboard - 设计需求文档

> **面向设计师** | 版本 1.0 | 2026-02-01

---

## 一、产品概述

开发一个个人工作 Dashboard，通过可爱的公仔动画展示 AI 助手（Monday）的工作状态，并用类似 Trello 看板的方式展示任务、创意、项目信息。

**核心价值**：让设计师/创业者一眼看懂 AI 在做什么、还有什么待办。

## 三、公仔动画设计需求

### 3.1 四种工作状态

| 状态 | 视觉表现 | 动画效果 |
|------|----------|----------|
| **空闲 (Idle)** | 公仔安静待着 | 轻微呼吸效果、偶尔眨眼 |
| **思考 (Thinking)** | 公仔若有所思 | 眼睛闪烁、头顶问号/灯泡 |
| **执行 (Executing)** | 公仔忙碌中 | 敲键盘/移动/发光效果 |
| **错误 (Error)** | 公仔出问题了 | 警示色闪烁、身体抖动 |

### 3.2 设计要求

- **风格**：可爱、友好、非威胁性（参考 Notion 风格 emoji 或 LINE Friends）
- **尺寸**：支持小(sm/32px)、中(md/64px)、大(lg/128px) 三档
- **动画帧率**：60fps 流畅动画
- **颜色**：与主题色保持一致，有状态颜色区分

**状态颜色参考**：
- 空闲：🟢 绿色/蓝色
- 思考：🟡 黄色/橙色
- 执行：⚡ 蓝色/紫色
- 错误：🔴 红色

### 4.2 卡片设计

**卡片内容**：
- 标题（一行）
- 标签（优先级 P0/P1/标签）
- 状态图标（已完成/进行中）
- 悬停时显示完整描述

**卡片悬停交互**：
```
鼠标悬停 → 显示 Tooltip 卡片详情
         ├── 完整描述（多行）
         ├── 创建时间
         ├── 备注数量
         └── 相关标签
```

**点击交互**：
```
点击卡片 → 打开 Modal 对话框
         ├── 标题 + 完整内容
         ├── 优先级标签
         ├── 备注编辑区
         └── 关闭按钮
```

### 4.3 视觉规范

- **卡片圆角**：8px
- **卡片间距**：12px
- **阴影**：悬浮时加深
- **标签颜色**：
  - P0：🔴 红色
  - P1：🟡 黄色
  - P2：🔵 蓝色
  - P3：⚪ 灰色

## 八、技术约束（设计师需知）

1. **框架**：React 18 + TailwindCSS
2. **动画库**：Framer Motion（推荐）或 CSS 动画
3. **响应式**：移动端适配（看板可改为横向滚动）
4. **图标库**：Lucide React（保持风格统一）
