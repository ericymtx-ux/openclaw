# Ideas - 灵感管理

用于记录和管理用户的灵感/想法。

## 使用方法

当用户输入灵感时：

1. **创建新文件**
   - 文件路径：`ideas/<概述>-<YYYY-MM-DD>.md`
   - 文件名格式：灵感的简短概述 + 日期

2. **文件内容模板**
   ```markdown
   # <灵感概述> - <YYYY-MM-DD>
   
   ## 原文
   <用户输入的原始内容>
   
   ## 总结
   <你对灵感的精简总结>
   
   ## TODO
   - [ ] <可执行的行动项1>
   - [ ] <可执行的行动项2>
   - [ ] <需要进一步思考的问题>
   
   ---
   *创建时间：<YYYY-MM-DD HH:mm:ss>*
   ```

## 目录结构

```
ideas/
├── README.md                    (本文档)
├── index.csv                    (灵感索引，便于搜索)
├── search-engine/               (检索工具开发区)
│   └── ...
├── <灵感概述>-<YYYY-MM-DD>.md   (灵感文件)
├── <另一个灵感概述>-<YYYY-MM-DD>.md
└── ...
```

## 索引文件 (index.csv)

每次创建新灵感时，同步更新 `index.csv`，包含：
- `filename` - 文件名
- `date` - 日期
- `summary` - 概述
- `tags` - 标签
- `created_at` - 创建时间

## 检索工具

后续将在 `search-engine/` 目录下开发检索功能，支持：
- 按关键词搜索灵感
- 按日期筛选
- 语义化检索

## 注意事项

- 灵感的"概述"要简洁，控制在 5-10 个字
- TODO 项要可执行，不要太抽象
- 每次灵感单独一个文件，不要合并
- 使用中文编写内容（除非用户用英文）
