# TODO 管理指南

## 添加 TODO 项

当有新的待办事项时：

1. **命名格式**：`[事项名称]_[日期].md`
   - 例如：`opencode-team修复_2026-02-01.md`
   - 日期格式：YYYY-MM-DD

2. **文件模板**：

```markdown
# [事项名称]

**创建日期**: YYYY-MM-DD
**状态**: pending | in_progress | completed
**负责人**: 

## 需求描述

[详细描述要解决的问题]

## 任务拆解

### 阶段 1: [阶段名称]
- [ ] 子任务 1
- [ ] 子任务 2

### 阶段 2: [阶段名称]
- [ ] 子任务 3
- [ ] 子任务 4

## 工作难点分析

### 技术难点
- [难点 1]: [描述和影响]

### 依赖关系
- [依赖 1]: [描述]

### 风险评估
- [风险 1]: [描述和应对措施]

## 验收标准

- [ ] 标准 1
- [ ] 标准 2

## 相关链接

- 相关 Issue:
- 相关 PR:
```

## 文件结构

```
TODO/
├── README.md              # 本文件
└── [事项名称]_[日期].md    # TODO 项文件
```

## 使用流程

1. 用户告知 TODO 事项
2. AI 创建 TODO 分析文件
3. 分析完成后开始执行
4. 完成后更新状态为 completed
